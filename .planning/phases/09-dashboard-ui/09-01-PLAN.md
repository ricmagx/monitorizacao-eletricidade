<task type="auto">
  <name>Task 2: SQLite reader functions em data_loader.py</name>
  <files>src/web/services/data_loader.py</files>
  <action>
Adicionar 4 funcoes ao ficheiro `data_loader.py` (sem modificar as funcoes existentes):

1. `load_consumo_sqlite(location_id: str, engine: Engine) -> list[dict]`:
   - SELECT de consumo_mensal WHERE location_id, ORDER BY year_month
   - Retorna lista de dicts com keys: year_month, total_kwh, vazio_kwh, fora_vazio_kwh
   - Retorna [] se sem dados
   - Usar pattern SQLAlchemy Core: `select(consumo_mensal).where(...)` com `row._mapping`

2. `build_analysis_from_sqlite(location_id: str, engine: Engine) -> dict | None`:
   - SELECT de comparacoes WHERE location_id, ORDER BY year_month
   - Para cada row: `json.loads(row.top_3_json)` e `json.loads(row.current_supplier_result_json)`
   - Construir dict com:
     - `"history"`: lista de dicts com year_month, top_3, current_supplier_result
     - `"history_summary"`: derivado do ultimo mes — latest_top_3, latest_current_supplier_result, latest_saving_vs_current_eur (calculado como current_cost - best_cost do top_3[0])
   - Retorna None se sem dados
   - CUIDADO: top_3_json ou current_supplier_result_json podem ser None — tratar com fallback para [] e {} respectivamente

3. `load_custos_reais_sqlite(location_id: str, engine: Engine) -> dict`:
   - SELECT de custos_reais WHERE location_id
   - Retorna dict {year_month: custo_eur} — formato compativel com o que build_custo_chart_data() espera
   - Retorna {} se sem dados

4. `get_freshness_from_sqlite(location_id: str, engine: Engine) -> dict`:
   - SELECT MAX(cached_at) de comparacoes WHERE location_id
   - Calcular days_ago e is_stale (threshold 40 dias) — mesma logica de get_freshness_info()
   - Se sem comparacoes, retornar {"days_ago": None, "is_stale": True, "generated_at": None}
