Modify `src/web/app.py` to:

1. Add a lifespan context manager that calls `metadata.create_all(engine)` as a safety net for local dev (Docker uses Alembic via entrypoint.sh):
```python
from contextlib import asynccontextmanager
from src.db.engine import engine
from src.db.schema import metadata

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Safety net para dev local sem Docker — Alembic e o caminho principal
    metadata.create_all(engine)
    yield
```

2. Pass `lifespan=lifespan` to the FastAPI constructor:
```python
app = FastAPI(title="Monitorizacao Eletricidade", lifespan=lifespan)
```

3. Add the engine to app.state so routes can access it:
```python
app.state.db_engine = engine
```

4. Add a `/health` endpoint that checks DB connectivity:
```python
from fastapi.responses import JSONResponse
from sqlalchemy import text

@app.get("/health")
def health():
    try:
        with engine.connect() as conn:
            conn.execute(text("SELECT 1"))
        return JSONResponse({"status": "ok", "db": "connected"})
    except Exception as e:
        return JSONResponse({"status": "error", "db": str(e)}, status_code=503)
```

5. Fix `PROJECT_ROOT` computation for Docker compatibility. Currently `BASE_DIR.parent.parent` from `src/web/app.py` gives `src/` not project root in some layouts. Use:
```python
import os
PROJECT_ROOT = Path(os.environ.get("APP_ROOT", str(BASE_DIR.parent.parent)))
```
This keeps backward compatibility (env var defaults to computed path) while allowing Docker to set `APP_ROOT=/app` if needed.

Keep all existing router registrations unchanged.
