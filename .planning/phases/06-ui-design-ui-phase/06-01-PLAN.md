<objective>
Produzir o contrato visual e de interacção (UI-SPEC.md) que guia todas as fases frontend (Phase 7, 8, 9, 10, 11) antes de qualquer implementação de código.

**Fase de design, não de código.** Output: 06-UI-SPEC.md com especificações completas e aprovadas.

**Âmbito:**
- Analisar o CSS e componentes Jinja2 existentes da v1.0 (style.css, partials/)
- Formalizar os tokens de design implícitos como sistema explícito
- Definir o layout v2.0 com novas secções (upload, configuração de locais)
- Especificar todos os contratos de interacção HTMX
- Definir copywriting completo (estados vazios, erros, confirmações)
- Definir contratos Chart.js para os dois gráficos
</objective>

<task id="06-01-T2">
<title>Definir design system completo e formalizar em UI-SPEC.md</title>
<read_first>
- .planning/REQUIREMENTS.md (requisitos UI-01 a UI-05)
- .planning/PROJECT.md (stack técnica: FastAPI + Jinja2 + HTMX + Chart.js, sem React/shadcn)
</read_first>
<action>
Com base na análise do task anterior:
1. Formalizar spacing scale com tokens nomeados (xs=4px, sm=8px, md=16px, lg=24px, xl=32px, 2xl=48px, 3xl=64px)
2. Definir tipografia: body 16px/400, label 13.6px/400, h2 19.2px/400, h1 24px/400 — apenas pesos 400 e 600
3. Classificar cores por papel semântico (dominant 60%, secondary 30%, accent 10%, success/warning/danger)
4. Reservar accent (#0d6efd) exclusivamente para CTAs primários (3 botões identificados)
5. Definir Component Inventory: existentes (estado v2.0) + novos por fase
6. Definir estrutura de layout: header → selector bar → banner → grid 2col → upload → ranking → footer
7. Definir breakpoints: >=769px grid 2col, <=768px stack vertical, max-width 1200px centrado
8. Definir Interaction Contract: selector HTMX, upload XLSX/PDF, formulário de local, banner condicional
9. Definir Copywriting Contract: todos os CTAs, empty states, erros, confirmações, mensagens de frescura
10. Definir Chart.js Contract: gráfico consumo (bar empilhado), gráfico custo (bar lado-a-lado), cores, legendas
</action>
